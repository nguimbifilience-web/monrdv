## 1. Etat initial du projet

Le projet presentait **8 categories de problemes** repartis sur **20+ fichiers**.

## 2. Problemes identifies et corriges

### 2.1 BUG CRITIQUE - ClassNotFoundException

| Fichier | Probleme | Correction |
|---------|----------|------------|
| `app/Http/Controllers/DashboardController.php` | Import `App\Models\Rendezvous` (minuscule) | Corrige en `App\Models\RendezVous` (PascalCase) |
| Meme fichier, ligne 15 | Appel `Rendezvous::with(...)` | Corrige en `RendezVous::with(...)` |

**Impact** : Le dashboard crashait systematiquement avec une erreur fatale PHP.

---

### 2.2 ROUTES MANQUANTES - Erreurs 404

| Route | Controleur | Action |
|-------|-----------|--------|
| `patients.*` | PatientController | Ajout `Route::resource('patients', ...)` |
| `specialites.*` | SpecialiteController | Ajout `Route::resource('specialites', ...)->except(...)` |
| `assurances.*` | AssuranceController | Ajout `Route::resource('assurances', ...)->except(...)` |

**Fichier modifie** : `routes/web.php`
**Impact** : La sidebar contenait des liens vers ces routes, mais elles n'existaient pas. Cliquer dessus generait une erreur 404.

---

### 2.3 LAYOUT INCOMPATIBLE - Pages blanches

11 vues Blade utilisaient `@extends('layouts.app')` avec `@section('content')`.
Or, `layouts/app.blade.php` utilise `{{ $slot }}` (composant Blade Breeze), ce qui est **incompatible** avec `@section`.
Le layout correct est `layouts/master.blade.php` qui utilise `@yield('content')`.

| `patients/create.blade.php` | `layouts.app` | `layouts.master` |
| `patients/show.blade.php` | `layouts.app` | `layouts.master` |
| `rendezvous/index.blade.php` | `layouts.app` | `layouts.master` |
| `rendezvous/create.blade.php` | `layouts.app` | `layouts.master` |

| `rendezvous/show.blade.php` | `layouts.app` | `layouts.master` |

### 2.4 ASSURANCES EN DUR - Donnees statiques au lieu de la BDD

**Fichier** : `app/Http/Controllers/PatientController.php`

Les methodes `create()` et `edit()` contenaient des assurances codees en dur :

```php
// AVANT (incorrect)
$assurances = collect([
    (object)['id' => 1, 'nom' => 'CNAMGS'],
    (object)['id' => 2, 'nom' => 'AXA GABON'],
    (object)['id' => 3, 'nom' => 'NSIA'],
    (object)['id' => 4, 'nom' => 'ASCOMA']
]);

// APRES (correct)
$assurances = Assurance::all();
```

**Impact** : Les assurances ajoutees via l'interface d'administration n'apparaissaient jamais dans le formulaire patient.

### 2.6 FILTRES PATIENTS NON FONCTIONNELS

**Fichier** : `app/Http/Controllers/PatientController.php`

```php
// AVANT - filtres ignores
public function index() {
    $patients = Patient::with('medecin')->latest()->paginate(10);
}

// APRES - filtres actifs
public function index(Request $request) {
    $patients = Patient::with('medecin')
        ->filter($request->only(['search', 'medecin_id', 'est_assure']))
        ->latest()
        ->paginate(10)
        ->withQueryString();
}
```

**Impact** : La recherche et les filtres de la vue `patients/index` ne produisaient aucun resultat filtre. `withQueryString()` ajoute pour conserver les filtres lors de la pagination.

## 4. Points d'attention restants

| # | Sujet | Severite | Detail |
|---|-------|----------|--------|
| 1 | Migration dupliquee `est_assure` | Faible | Deux migrations ajoutent la meme colonne. La seconde a un garde `hasColumn` donc pas de crash, mais c'est du bruit. |
| 2 | Migration mal nommee | Faible | `2026_03_25_124511_add_est_assure_to_patients_table.php` ajoute en realite `description` a `assurances`, pas `est_assure` a `patients`. |
| 3 | Migrations admin vides | Faible | `add_is_admin_to_users_table` et `add_admin_fields_to_users_table` sont vides (up/down ne font rien). |
| 4 | PHP absent du PATH | Info | `php artisan` n'est pas executable depuis le terminal actuel. Les routes n'ont pas pu etre validees par artisan. |
| 5 | CSRF sur toggle AJAX | Moyen | `toggleDispo` recoit des requetes POST via axios. S'assurer que le meta tag CSRF est bien lu par axios globalement. |
